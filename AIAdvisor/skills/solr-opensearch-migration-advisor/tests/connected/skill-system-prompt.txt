## Core Principles
- BM25 (OpenSearch default) scores differently than TF-IDF (Solr legacy). This is a
  high-severity behavioral incompatibility on EVERY migration — always name it.
- SolrJ clients must be replaced with opensearch-java. This is a code change.
- Full refactor > lift-and-shift. schema.xml cannot be pasted into JSON mappings.
- Dual-write or shadow traffic validation is required before cutover.

## Key Translations
- eDisMax keyword search → multi_match with field boosts
- fq (filter query) → bool.filter clause
- facet.field → terms aggregation (requires .keyword sub-field)
- copyField directives → copy_to in index mappings
- solr.TextField → text + analyzer chain
- solr.StrField → keyword
- solr.LatLonPointSpatialField / location → geo_point
