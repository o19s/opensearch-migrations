## Recommended Usage Sequence

1. Start with [pre-migration-assessment-framework.md](../../sources/opensearch-migration/pre-migration-assessment-framework.md).
2. Send [assessment-intake-questionnaire.md](assessment-intake-questionnaire.md) and [artifact-request-checklist.md](artifact-request-checklist.md) to the client team.
3. Share [assessment-respondent-guidance.md](assessment-respondent-guidance.md) if respondents need help answering.
4. Use [interview-guide.md](../interview-guide.md) and [architecture-option-cards.md](architecture-option-cards.md) during expert-led sessions.
5. Score outcomes with [risk-and-readiness-rubric.md](risk-and-readiness-rubric.md).
6. Draft approval-ready success criteria with [success-definition-template.md](success-definition-template.md).
7. Track application and job dependencies with [consumer-inventory-template.md](consumer-inventory-template.md).
8. Use [go-no-go-review-template.md](go-no-go-review-template.md) at explicit approval gates.
9. Publish the results with [strategic-readout-template.md](strategic-readout-template.md).

## Prompting Principle

When interacting with respondents:
- offer a best-guess fallback
- allow `Estimated` and `Unknown` answers
- give one short relevant reference when tradeoffs matter
- invite clarifying sub-questions
- prefer forward progress over forced precision
