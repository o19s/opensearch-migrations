# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

AIAdvisor is a Claude/Bedrock-powered **Solr-to-OpenSearch migration advisor** skill. It teaches an LLM the domain knowledge needed to translate Solr configurations, queries, and clients into OpenSearch equivalents. The primary deliverable is a system prompt (`skill-system-prompt.txt`) evaluated against real infrastructure using [promptfoo](https://promptfoo.dev/).

## Key Domain Translations (for prompt work)

| Solr | OpenSearch |
|------|------------|
| eDisMax | `multi_match` |
| Facets | Aggregations |
| `tf-idf` scoring | BM25 |
| SolrJ client | `opensearch-java` |
| SolrCloud | OpenSearch clusters |
| `copyField` | `copy_to` |
| `fieldType` | `mappings` |
| `geo_distance` range | `geo_point` + `geo_distance` filter |

Express/YOLO mode: generates complete specs immediately and marks assumptions with `[ASSUMED: ...]`.

## Test Infrastructure

Two test suites, different scopes:

| Suite | Path | Purpose |
|-------|------|---------|
| `skill-smoke` | `tests/skill-smoke/` | Step-by-step skill loading + promptfoo adoption |
| `connected` | `tests/connected/` | Full E2E with Docker: Solr seeding, LLM eval, optional migration |

### Skill Smoke Tests (start here)

```bash
cd skills/solr-opensearch-migration-advisor/tests/skill-smoke

./run_smoke.sh                  # all 5 steps
./run_smoke.sh --step 2         # single step
./run_smoke.sh --steps 1,2      # subset
./run_smoke.sh --step 5 --solr-url http://localhost:38983  # live Solr
./run_smoke.sh --step 5 --start-docker                     # spin up Docker first
```

Five steps in order:

| Step | What it does | Key assertion |
|------|-------------|---------------|
| 1 | Bare metal AWS CLI call, no skill | LLM responds (non-empty) |
| 2 | Load nuggets 1→2→3, run 3 assertions | 1/3 → 2/3 → 3/3 pass |
| 3 | promptfoo with **inline** prompt (1 nugget in yaml) | 1/3 pass (expected fails documented) |
| 4 | promptfoo with **external** `file://` prompt (all nuggets) | 3/3 pass |
| 5 | Live Solr: YOLO mode + multi-turn interactive | EXPRESS MODE, BM25, interactive Q&A |

The nuggets (`nuggets/nugget-0N-*.txt`) each unlock one assertion:
- `nugget-01-express-mode.txt` → `EXPRESS MODE` banner in output
- `nugget-02-bm25.txt` → `BM25 vs TF-IDF` phrase in output
- `nugget-03-translations.txt` → `opensearch-java` in output

`skill-assembled.txt` (generated by `--step 4`, gitignored) is the concatenated full skill used by `promptfooconfig-external.yaml`.

### Connected Tests (full E2E)

```bash
cd skills/solr-opensearch-migration-advisor/tests/connected

# Default: connectivity checks + LLM skill evaluation (~2-3 min)
./run_connected_tests.sh

# Full migration pipeline: export → transform → bulk load → verify (~5-10 min)
./run_connected_tests.sh --migrate

# Keep containers running after tests (for re-running verification)
./run_connected_tests.sh --no-teardown

# Re-run verification only (after --no-teardown)
./verify_migration.sh --solr-port 38983 --os-port 39200

# Re-run promptfoo eval directly (requires running services)
npx promptfoo eval
npx promptfoo view   # Open report UI
```

### Prerequisites

- Docker & docker compose
- Node.js 18+ (for `npx promptfoo`)
- AWS credentials with Bedrock access (for LLM eval assertions)

### AWS Credentials (for Bedrock)

Set one of:
- Environment variables: `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_SESSION_TOKEN`, `AWS_DEFAULT_REGION`
- AWS CLI profile: `aws configure` or `aws sso login`

### Service Ports (non-standard to avoid dev conflicts)

|---------|-----------|----------------|
| Solr 8 | 38983 | 8983 |
| OpenSearch 3.3 | 39200 | 9200 |
| ZooKeeper | 32181 | 2181 |


### Key Files

- **`skill-system-prompt.txt`** — The core LLM system prompt defining migration advisor behavior. This is the primary artifact being developed.
- **`promptfooconfig.yaml`** — Defines eval assertions against the skill. Uses two providers: HTTP (infrastructure health checks) and Bedrock (LLM quality assertions). Model: `amazon.nova-pro-v1:0`.
- **`run_connected_tests.sh`** — Orchestrates Docker startup, Solr seeding, promptfoo eval, optional migration, and teardown. Detects already-running containers to enable fast re-runs.
- **`test_helpers.sh`** — `assert_eq`, `assert_contains`, `assert_http_ok` functions with JUnit XML + plain text report generation.

### Promptfoo Eval Assertions (connected tests)

Infrastructure checks (HTTP provider):
- Solr health returns HTTP 200
- Solr doc count = 5 seeded TechProducts docs
- OpenSearch health returns HTTP 200

Skill quality checks (Bedrock provider, evaluated by `amazon.nova-micro-v1:0`):
- Output mentions BM25
- Output mentions `opensearch-java` client
- Output mentions `multi_match` (eDisMax translation)
- Output mentions `geo_point`
- Output uses `[ASSUMED: ...]` markers
- Output contains a JSON code block
